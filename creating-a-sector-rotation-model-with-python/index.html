Creating a Sector Rotation Model with Python


Sector rotation is an investment strategy that seeks to capitalize on the changing economic cycles by shifting portfolio allocations among different sectors of the economy. By identifying which sectors are likely to outperform at any given stage of the economic cycle, investors can optimize returns and manage risk more effectively. Python, with its powerful data analysis libraries and frameworks, offers an excellent toolkit for developing a systematic sector rotation model.

In this comprehensive guide, we will build a sector rotation model step-by-step using Python. We’ll cover data acquisition, processing, factor creation, ranking sectors, and backtesting the strategy. Whether you’re a quant enthusiast or a portfolio manager looking for automation, this walkthrough will equip you with the fundamentals required to construct a practical rotation model.

Understanding the Concept of Sector Rotation

Before diving into code, it is crucial to grasp what sector rotation entails and why it works. The economy tends to move through cycles: expansion, peak, contraction, and trough. Different sectors react differently depending on the phase:

  • Early expansion: Cyclical sectors like Technology, Consumer Discretionary tend to lead.
  • Mid to late expansion: Industrials and Materials often gain momentum.
  • Contraction and recession: Defensive sectors such as Utilities, Consumer Staples, and Healthcare outperform.
  • Recovery: Financials typically rebound strongly.

A rotation model aims to capture these transitions by shifting weight from underperforming sectors to those poised to outperform based on historical data and economic indicators.

    'XLB',  # Materials

]

# Download historical data
data = yf.download(sector_tickers, start="2010-01-01", end="2024-01-01")['Adj Close']

# Check the first few rows
print(data.head())

This code pulls adjusted close prices for nine sector ETFs over 14 years, providing a robust data sample to analyze trends and build the rotation model.

Step 2: Calculating Returns and Industry Metrics

Raw price data is not as informative as returns or momentum indicators that reflect sector performance trends. Commonly used metrics for sector rotation include:

  • Momentum: Rate of return over a specific period, e.g., 3 or 6 months.
  • Volatility: Standard deviation of returns helps assess risk.
  • Relative Strength: How a sector performs relative to the market.

Letâ€™s calculate monthly returns and momentum indicators for each sector:

# Resample daily data to monthly frequency, taking last closing price of each month
monthly_prices = data.resample('M').last()

# Calculate monthly returns (percentage change)
monthly_returns = monthly_prices.pct_change().dropna()

# Define rolling momentum window in months
momentum_window = 3

# Calculate momentum as cumulative return of last 'momentum_window' months
momentum = monthly_returns.rolling(window=momentum_window).apply(lambda x: (1 + x).prod() - 1)

print(momentum.tail())

This rolling calculation reveals which sectors have been gaining momentum recently, a key predictive factor for future outperformance.

Step 3: Ranking and Selecting Sectors

With momentum values computed, the next step is ranking sectors from strongest to weakest. Usually, a rotation model goes long the top sectors with highest momentum and avoids or shorts the laggards.

We can create a function that ranks sectors, selects the top N, and generates position weights:

def rank_sectors(momentum_df, top_n=3):
    """
    Rank sectors by momentum and select top N sectors.
    Returns weights for each sector (equal weight among selected).
    """
    ranks = momentum_df.rank(axis=1, ascending=False)
    top_sectors = ranks <= top_n
    
    # Assign equal weights to top sectors, zero to others
    weights = top_sectors.div(top_sectors.sum(axis=1), axis=0).fillna(0)
    return weights

# Generate sector weights based on momentum rankings
weights = rank_sectors(momentum)

print(weights.tail())

This produces a DataFrame indicating which sectors to hold each month, with allocation weights normalized to sum to 1.

Step 4: Backtesting the Sector Rotation Strategy

Backtesting provides a view of how the model would have performed historically if followed faithfully. We apply sector weights to next month’s returns to calculate portfolio-level returns:

# Align weights with returns for proper shifting
shifted_weights = weights.shift(1).dropna()
aligned_returns = monthly_returns.loc[shifted_weights.index]

# Calculate portfolio returns by weighted sum of sector returns
portfolio_returns = (shifted_weights * aligned_returns).sum(axis=1)

# Calculate cumulative return over backtest period
cumulative_returns = (1 + portfolio_returns).cumprod() - 1

import matplotlib.pyplot as plt

plt.figure(figsize=(12, 6))
plt.plot(cumulative_returns, label='Sector Rotation Portfolio')
plt.title('Backtest: Sector Rotation Model Performance')
plt.xlabel('Date')
plt.ylabel('Cumulative Return')
plt.legend()
plt.grid(True)
plt.show()

This visualization highlights the potential gains and drawdowns, allowing you to compare against benchmarks like the S&P 500 to gauge effectiveness.

Step 5: Incorporating Risk Management Measures

Sector rotation models are more effective when combined with risk controls. Some common techniques include:

  • Volatility targeting: Adjust exposure based on recent market volatility to smooth returns.
  • Drawdown limits: Temporarily reduce exposure after large losses.
  • Diversification hedges: Allocate a portion of the portfolio to defensive sectors or cash during downturns.

Let’s add a volatility-based scaling factor to our weights to demonstrate:

# Calculate rolling volatility of portfolio returns
vol_window = 6
rolling_vol = portfolio_returns.rolling(window=vol_window).std()

# Define target volatility (e.g., 5% annualized, converted to monthly)
target_vol = 0.05 / (12 ** 0.5)

# Compute scaling factors
scaling_factors = target_vol / rolling_vol
scaling_factors = scaling_factors.clip(upper=1)  # Limit scaling to 1

# Apply scaling to portfolio returns
scaled_returns = portfolio_returns * scaling_factors

# Recalculate cumulative returns with scaling
scaled_cumulative_returns = (1 + scaled_returns.dropna()).cumprod() - 1

plt.figure(figsize=(12, 6))
plt.plot(scaled_cumulative_returns, label='Volatility-Adjusted Sector Rotation')
plt.legend()
plt.title('Sector Rotation with Volatility Targeting')
plt.xlabel('Date')
plt.ylabel('Cumulative Return')
plt.grid(True)
plt.show()

This approach reduces allocations during turbulent times, potentially limiting drawdowns without sacrificing long-term growth.

Step 6: Enhancements and Next Steps

The model developed here is a foundational framework. There are many directions for refinement:

  • Factor blending: Incorporate economic indicators (yield curve, PMI) or sentiment data to improve timing.
  • Machine Learning: Use classification or ensemble models to predict sector leaders dynamically.
  • Rebalancing frequency: Test weekly or daily rotation versus monthly to capture shorter trends.
  • Transaction costs: Model impact of trading fees and slippage to gauge real-world viability.
  • Optimization: Use mean-variance or risk parity techniques to improve weight allocation rather than equal-weighting.

Building on this foundation, one can tailor the sector rotation model to align with specific risk tolerances, investment horizons, or thematic views.
